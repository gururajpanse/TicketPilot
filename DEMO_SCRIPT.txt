[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at our system architecture. (pause) 

Every incoming request begins at the Security Checkpoint. If an email is from an unauthorized domain or contains prompt injections, it is immediately diverted to the Security Event Handler, which bypasses the orchestrator to output a blocked event. 

Clean and authorized requests are handed to the Triage Orchestrator Agent. The Orchestrator delegates specific tasks to six specialized sub-agents:

First, the Ingestion Agent parses and normalizes details into the shared workflow context state.

Second, the Classification Agent assigns the IT category and determines ticket severity from P1 to P4.

Third, the Escalation Agent instantly alerts on-call engineers for critical P1 events.

Fourth, the Deduplication Agent queries the MCP server's active outages tool to cluster incoming duplicate tickets during major storms.

Fifth, the Resolution Agent searches the MCP runbook database to apply standard automated fixes and drafts knowledge base articles.

And sixth, the Routing Agent assigns any unresolved tickets directly to human specialist queues.

After the orchestrator completes, our post-orchestrator router decides the next node. If a ticket status is NEEDS_APPROVAL, the flow routes to the Human Approval Node, yielding a RequestInput that pauses execution. Once the operator inputs their response, the workflow resumes and passes to the Final Output Node to deliver the structured JSON response.
