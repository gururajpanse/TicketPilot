[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. (pause) Every request begins at the Security Checkpoint. This is a deterministic node that performs prompt injection checks, scrubs credit cards or passwords, and blocks unauthorized external emails.
If the request is clean, it is handed over to the Triage Orchestrator. The Orchestrator does not work alone. Instead, it delegates specific tasks to our six sub-agents:
First, the Ingestion Agent parses and normalizes the request details, saving them to the shared workflow state.
Next, the Classification Agent determines the IT category and assigns a severity from P1 to P4.
The Escalation Agent immediately handles P1 events, bypassing the normal queue to page on-call engineers via our PagerDuty tool.
If it is a standard issue, the Deduplication Agent queries the MCP server for active incident storms to cluster duplicates.
The Resolution Agent searches the runbook database to apply standard automated resolutions, drafting knowledge base articles for new solutions.
And finally, the Routing Agent assigns unresolved tickets to specialist queues like the Network or Access teams.
At the end, if a ticket requires operator verification, it enters the Human-in-the-Loop approval node, pausing the workflow until approved.
